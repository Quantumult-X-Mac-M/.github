## 🔍 Overview

<a href="#quantumultX">Quantumult X</a> is an advanced network traffic management and proxy tool that gives technically sophisticated users granular control over how network requests are routed, filtered, modified, and monitored — extending far beyond basic VPN or proxy functionality into scriptable, rule-based network behavior customization.

The <a href="#quantumultX">rule-based traffic routing</a> directs network requests through different paths based on configurable rules — domain matching, IP range matching, geographic location, and other criteria determine whether specific traffic routes through a proxy, connects directly, or is blocked entirely. This rule system enables precise control over network behavior on a per-domain or per-service basis rather than the all-or-nothing routing that simpler VPN tools provide. The <a href="#quantumultX">script-based automation</a> allows JavaScript scripts to intercept, modify, and respond to network requests programmatically — extending Quantumult X's behavior far beyond static rules into dynamic processing that can modify request headers, transform response content, implement custom logic for specific services, and automate complex network interaction patterns.

The <a href="#quantumultX">request rewrite capability</a> modifies requests and responses according to configured rewrite rules — redirecting requests, modifying headers, blocking specific request patterns, and altering network behavior at the request level for specific use cases that require this granular intervention. The <a href="#quantumultX">multiple protocol support</a> covers numerous proxy protocols including Shadowsocks, VMess, Trojan, and others — providing compatibility with the wide range of proxy server configurations that different services and providers use.</p>

<a href="#quantumultX">Traffic monitoring</a> provides real-time visibility into network requests passing through Quantumult X — viewing which domains are being accessed, which rules are matching, and how traffic is being routed for debugging rule configurations and understanding network behavior. <a href="#quantumultX">Configuration file management</a> handles the complex configuration files that define rules, servers, and behavior — importing, editing, and switching between different configuration profiles for different use cases or network contexts.

<a href="#quantumultX">Filter and block rules</a> prevent specific traffic from reaching its destination — blocking ad-serving domains, tracking domains, or any other traffic pattern that should be filtered from normal network flow. <a href="#quantumultX">DNS configuration control</a> manages how domain name resolution is handled — directing DNS queries through specific resolvers or applying custom DNS behavior for the routing rules to work correctly with domain-based matching.</p>
